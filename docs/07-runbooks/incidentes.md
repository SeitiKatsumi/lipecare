# Runbook: Incidentes

## Exemplos De Incidente

- vazamento de dados;
- acesso indevido a paciente;
- banco indisponivel;
- storage expondo arquivo privado;
- token comprometido;
- deploy quebrado;
- erro em integracao de WhatsApp ou IA.

## Resposta Inicial

1. Conter o problema.
2. Preservar evidencias.
3. Registrar horario e impacto.
4. Rotacionar credenciais se necessario.
5. Comunicar responsaveis.
6. Aplicar correcao.
7. Fazer postmortem.

## Postmortem

Registrar:

- o que aconteceu;
- impacto;
- causa raiz;
- acao corretiva;
- prevencao futura;
- responsaveis;
- prazo.
